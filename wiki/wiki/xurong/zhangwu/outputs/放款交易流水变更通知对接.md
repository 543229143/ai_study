# 放款交易流水变更通知对接说明

来源：[[../sources/账务对外常用解释|账务对外常用解释]]

## RabbitMQ 配置

| 项 | 值 |
|---|---|
| ExchangeType | DIRECT |
| Exchange | `exchange.direct.lcs-service` |
| Queue | `queue.lcs-{消费服务名}.tranProcChange` |
| BindKey | `key.lcs.tranProcChange` |
| 消息体类 | `PilotTranProcChangePushDto` |

`{消费服务名}` 替换为你的服务名即可。

## 触发场景

仅在**放款中**及**放款交易流水表终态**时推送：

| 状态 | 说明 |
|------|------|
| `DB_DRAW_PROCESSING` | 放款中 |
| `DB_PUTOUT_SUCCESS` | 放款成功 |
| `DB_PUTOUT_FAIL` | 放款失败 |
| `DB_PUTOUT_REJECT` | 放款拒绝 |
| `DB_APPLY_CANCEL` | 申请取消 |
| `DB_LOAN_CANCEL` | 取消贷款 |

## 消息体字段

| 字段 | 类型 | 必传 | 说明 |
|------|------|------|------|
| custNo | String | 是 | 客户号 |
| acctNo | String | 是 | 账号 |
| fundCode | String | 是 | 机构编码 |
| applyReqNo | String | 是 | 申请单号（PilotTranProcDb.applyNo） |
| tranProcDbNo | String | 是 | 放款交易流水号 |
| loanNo | String | 否 | 借据号（仅放款成功时有） |
| channelId | String | 是 | 授信申请渠道号 |
| loanSource | String | 是 | 借款申请渠道号 |
| loanAmt | BigDecimal | 否 | 借款金额（元，部分 H5 机构可能没有） |
| term | Integer | 否 | 申请期数（部分 H5 机构可能没有） |
| previousStatus | String | 是 | 变更前状态 |
| currentStatus | String | 是 | 变更后状态（当前状态） |
| priceType | String | 否 | 定价类型（IRR24 / IRR36 / IRR24_PRIVILEGE / IRR24_AMC） |
| loanApplyType | String | 是 | 借款申请类型 |
| pushTime | String | 是 | 推送时间（yyyy-MM-dd HH:mm:ss:SSS） |

## 对接要点

1. **幂等处理**：以 `tranProcDbNo` 作为唯一键做去重
2. **状态机**：关注 `previousStatus → currentStatus` 的流转
3. **loanNo 为空处理**：非放款成功的通知没有借据号，需按 `tranProcDbNo` 关联
4. **部分字段可能缺失**：`loanAmt`、`term` 对 H5 机构可能为空，消费方需做容错
