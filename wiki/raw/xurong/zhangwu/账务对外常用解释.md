## 发起借款前权益数据获取

需调用借款试算dubbo服务获取

**接口基本信息**

所在服务: lcs\-service

服务名: PilotPutOutFacade\#putOutCalc



**是否支持权益判断（即是否展示权益）：**

入参：supportPrivilege=True

出参：pilotPrivilegeNew\.showPrivilege=True



**选择24\+权益试算，机构支持权益（返回的还款计划数据是带权益的）：**

入参：supportPrivilege=True **\&\& **selectPrivilege=True

出参：pilotPrivilegeNew\.showPrivilege=True



**选择24\+权益试算，但机构不支持权益（返回的还款计划数据是不带权益的）：**

入参：supportPrivilege=True **\&\& **selectPrivilege=True

出参：pilotPrivilegeNew\.showPrivilege=False



**支持权益但未选择权益试算（返回的还款计划数据是不带权益的）**

入参：supportPrivilege=True **\&\& **selectPrivilege=False

出参：pilotPrivilegeNew\.showPrivilege=True



**平台不支持权益（返回的还款计划数据是不带权益的）**

入参：supportPrivilege=False

出参：pilotPrivilegeNew\.showPrivilege=False

## 发起借款后权益数据查询

### 2\.1 权益状态

借款单是否有权益，需判断权益流水表是否存在数据。

关联条件

lcs\.pilot\_tran\_proc\_db\.tran\_proc\_db\_no = lcs\.pilot\_privilege\_tran\_proc\.tran\_proc\_db\_no



权益开通成功后，会落地权益订单\(lcs\.pilot\_privilege\_order\)和权益还款计划\(lcs\.pilot\_privilege\_plan\)。

权益开通成功且权益订单生成成功后，更新权益流水表上的权益订单号\(lcs\.pilot\_privilege\_tran\_proc\.privilege\_order\_no\)，该字段在开通成功时会有值。



权益流水表和其关联关系:

lcs\.pilot\_privilege\_tran\_proc\.privilege\_order\_no = lcs\.pilot\_privilege\_order\.order\_no

lcs\.pilot\_privilege\_tran\_proc\.privilege\_order\_no = lcs\.pilot\_privilege\_plan\.order\_no



可行权时，数据对应状态说明：

1. lcs\.pilot\_privilege\_order\.status满足'RP','OD','FP'其一

2. lcs\.pilot\_privilege\_order\.date\_expired 不为空，且当前时间小于该日期

|状态|取数逻辑|备注|
|---|---|---|
|开通中|lcs\.pilot\_privilege\_tran\_proc\.status = 'PROCESS'||
|开通失败|lcs\.pilot\_privilege\_tran\_proc\.status = 'FAIL'||
|使用中（开通成功，可行权）<br>|1. lcs\.pilot\_privilege\_order\.status满足'RP','OD','FP'其一<br>2. lcs\.pilot\_privilege\_order\.date\_expired 不为空，且当前时间小于该日期<br>|可行权时，数据对应状态说明：<br>1. lcs\.pilot\_privilege\_order\.status满足'RP','OD','FP'其一<br>2. lcs\.pilot\_privilege\_order\.date\_expired 不为空，且当前时间小于该日期|
|已过期<br>|1. lcs\.pilot\_privilege\_order\.date\_expired 不为空，且当前时间大于该日期<br>2. lcs\.pilot\_privilege\_order\.status满足'RP','OD','FP'其一||
|退款中<br>|lcs\.pilot\_privilege\_order\.status = 'RFP'<br>|如渠道没有退款中状态，可按调整前逻辑去映射处理：<br>调整前的使用中（开通成功，可行权）和已过期取数逻辑：<br><br>使用中（开通成功，可行权）：<br>1. lcs\.pilot\_privilege\_order\.status满足'RP','OD','FP','RFP'其一<br>2. lcs\.pilot\_privilege\_order\.date\_expired 不为空，且当前时间小于该日期<br><br>已过期：<br>1. lcs\.pilot\_privilege\_order\.date\_expired 不为空，且当前时间大于该日期<br>2. lcs\.pilot\_privilege\_order\.status满足'RP','OD','FP','RFP'其一<br><br>调整原因：<br>如出现RFP（退款中）状态，说明权益已扣款成功。权益过期时间（lcs\.pilot\_privilege\_order\.date\_expired）一定有值。|
|已退款|lcs\.pilot\_privilege\_order\.status = 'CP'||
|无权益申请记录|lcs\.pilot\_privilege\_tran\_proc 无数据||
|开通成功，但暂不可使用（机构是先付后享模式，需主动支付/批扣成功后才可以行权）|1. lcs\.pilot\_privilege\_order\.status为'RP','OD'其一<br>2. lcs\.pilot\_privilege\_order\.date\_expired 为空<br>|·<br>|
|权益开通成功后，取消（未主动支付/批扣成功）|lcs\.pilot\_privilege\_order\.status = 'CLO'<br>||

|无搭售产品流水<br>|返回原始借据及还款计划（未经加工）|ORIGINAL\_NO\_ADDON<br>|
|有搭售产品流水|根据流水状态处理|\-|



**第三步：根据流水状态处理**



|流水状态|处理逻辑|返回状态|
|---|---|---|
|FAIL（开通失败）<br>|返回原始借据及还款计划（未经加工）|ORIGINAL\_PROCESS\_FAIL|
|PROCESS（开通中）<br>|返回空数据<br>|EMPTY\_DATA\_PROCESSING|
|SUCCESS（开通成功）<br>|查询搭售订单和还款计划，进行数据合并|MERGED\_DATA 或 EMPTY\_DATA\_PROCESSING（还款计划未落地成功）|



##### **数据合并逻辑**



\> **合并顺序**：先合并还款计划，再合并借据



###### **还款计划合并（以AMC搭售产品为例）**



1\) **费用项计算**

- 本金 = 借据还款计划本金

- 利息 = 借据还款计划利息

- 其他费用 = 借据还款计划其他费用\(不包含搭售产品费用\)

- 搭售产品费用1 = 搭售产品还款计划费用1

- 搭售产品费用2 = 搭售产品还款计划费用2

- 搭售产品费用3 = 搭售产品还款计划费用3

- 逾期费 = 借据还款计划逾期费 

- 总金额 = 以上汇总

- 赋值应还金额、实还金额、减免金额

2\) **还款日计算**

- 取借据/搭售还款计划的最小还款日

3\) **最早可还款日期计算**

- 和还款日计算联动,如还款日取的借据的\.最早可还款日期取借据的\.如最早可还款日期为空,使用还款日\.

4\) **状态计算**



|借据状态|搭售状态|合并状态|
|---|---|---|
|结清|结清|结清|
|待还|结清|待还|
|逾期|结清|逾期|
|结清|待还|待还|
|结清|逾期|逾期|
|待还|待还|待还|
|逾期|逾期|逾期|



5\) **结清时间计算**

- 前置条件：合并状态为结清

- 取借据、搭售中最晚的结清时间

6\) **逾期天数计算**

- 结清：结清时间 \- 还款日

- 逾期/待还：当前时间 \- 还款日，结果小于0时取0

7\) **是否提前结清标识\(计算逻辑同借据还款计划\)**



**逻辑说明**：

- 前置条件：借据状态为结清（FP）且搭售产品状态也为结清（FP）

- 判断是否存在同一天内多期计划结清

- 找出最早应该结清的期次（根据应还日期与实际结清日期比较）

- 如果最后一期不是应该结清的起始期次，则认为提前结清

**计算步骤**：

1. 检查最后一期的状态是否为结清（FP）

2. 按结清日期分组，找出同一天结清的所有期次

3. 对于同一天结清的多期中，找出最早期次：

    - 根据该期的应还日与实际结清日比较

    - 若应还日 \> 结清日，则起始期 = 该期次；否则起始期 = 该期次 \+ 1

4. 如果最后一期的期次不等于计算出的起始期，则这些期次存在提前结清

5. 将所有 \>= 起始期的期次标记为提前结清（Y），其他标记为N

8\) **其他字段赋值**

- 均使用借据还款计划数据

###### **借据合并（以合并后的还款计划作为数据源）**



1\) **状态计算（status）**

- 如借据状态为退贷 → status = 借款取消（CP）

- 所有还款计划均已结清 → status = 结清（FP）

- 存在未结清期数且最近未结清期状态为待还 → status = 还款中（RP）

- 存在未结清期数且最近未结清期状态为逾期 → status = 逾期中（OD）

2\) **结清日期计算（dateSettle）**

- 仅当所有还款计划均已结清时，取最后一期的结清日期

3\) **逾期天数计算（overdueDays）**

- 取最近未结清期数的逾期天数

- 若无未结清期数，则取最后一期的逾期天数

4\) **最大逾期天数计算（maxOverdueDays）**

- 若计算出的逾期天数 \> 当前借据最大逾期天数（搭售产品没有最大逾期天数），则更新为该值

5\) **是否提前结清标识（isSettledEarly）**

- 仅当所有还款计划均已结清时，取最后一期的提前结清标识

6\) **最近还款日期计算（latestRepayDate）**

- 依次遍历所有还款计划（期数 \> 0）：

    - 若当期待还且还款日 \>= 当前日期，返回该期还款日

    - 若当期逾期，返回该期还款日

- 若无符合条件的期数，则返回最后一期的还款日
